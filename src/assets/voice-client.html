Usage Instructions

The Voice SDK Client allows you to customize call destinations and behavior using URL parameters. Simply add parameters to the URL to configure where your calls should be routed.

Basic Usage

Open the voice client with custom parameters in your browser:

https://YOUR_DOMAIN_NAME/voice-client.html?parameter=value

Routing Options

1. Route to TwiML App

Route calls to a specific TwiML Application.

https://YOUR_DOMAIN_NAME/voice-client.html?twimlAppSid=APxxxxxxxxx
  • twimlAppSid Required
    TwiML Application SID to route the call to (starts with AP)

2. Connect to an AI Assistant

Route calls to a Twilio AI Assistant with custom greetings and voices.

https://YOUR_DOMAIN_NAME/voice-client.html?assistantSid=ASxxxx&greeting=Hello!&voiceId=en-US-Journey-O
  • assistantSid Required
    Your AI Assistant SID (starts with AS)
  • greeting Optional
    Custom greeting message for the AI Assistant
  • voiceId Optional
    Voice ID for the assistant (e.g., en-US-Journey-O, en-US-Neural2-A)

3. Call a Phone Number

Route calls to a phone number in E.164 format.

https://YOUR_DOMAIN_NAME/voice-client.html?destinationType=phone&phoneNumber=+15551234567
  • destinationType Required
    Set to "phone" to route to a phone number
  • phoneNumber Required
    Phone number to call in E.164 format (e.g., +15551234567)

Note: Requires TWILIO_MOBILE_NUMBER configured in your environment for caller ID

4. Route to Flex

Route calls to a Twilio Flex workflow for agent handling.

https://YOUR_DOMAIN_NAME/voice-client.html?destinationType=flex
  • destinationType Required
    Set to "flex" to route to Flex workflow

Note: Requires FLEX_WORKSPACE_SID and FLEX_WORKFLOW_SID configured in your environment

5. Custom Routing

Use custom TwiML logic defined in your backend.

https://YOUR_DOMAIN_NAME/voice-client.html?destinationType=custom
  • destinationType Required
    Set to "custom" to use custom TwiML routing logic

Note: Extend the voice-sdk-call.protected.ts function to add custom TwiML logic

Configuration

No URL Parameters: If you open the voice client without any URL parameters, you can enter a phone number directly in the input field. The number must be in E.164 format (e.g., +15551234567). You can also use formats like +1-555-123-4567 or +1 555 123 4567, which will be automatically cleaned for validation.

With URL Parameters: You can pre-configure the call destination using URL parameters as shown in the examples above. When URL parameters are provided, they will override the input field and display the configured destination as read-only.

For more information about setup and configuration, see the README documentation in the project repository.
